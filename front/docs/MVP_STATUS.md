## ⚠️ MVP에서 부족한 부분

### 1. 페이지네이션 (중요도: 높음)
- 현재: limit만 있음
- 문제: 데이터가 많아지면 성능 이슈
- 해결: 페이지네이션 또는 무한 스크롤

### 2. 실제 백엔드 연동 (중요도: 매우 높음)
- 현재: Mock API (localStorage 기반)
- 문제: 실제 서비스 불가
- 해결: 백엔드 API 연동

### 3. 결제 시스템 (중요도: 높음)
- 현재: 후원/입찰은 데이터만 저장
- 문제: 실제 결제 없음
- 해결: 결제 API 연동 (토스페이먼츠, 아임포트 등)

### 4. 프로젝트 시작 알림 (중요도: 중간)
- 현재: 경매만 알림 있음
- 문제: 프로젝트 시작 알림 없음
- 해결: 프로젝트 시작 알림 추가

## 🎯 다음 개발 추천 (우선순위 순)

### 1순위: 페이지네이션 (필수)

**이유:**
- 데이터가 많아지면 필수
- 사용자 경험 향상
- 성능 최적화

**구현 방법:**
- 무한 스크롤 (추천) 또는 페이지네이션 컴포넌트
- Intersection Observer API 사용
- 로딩 상태 관리

**예상 시간:** 2-3시간

---

### 2순위: 프로젝트 시작 알림

**이유:**
- 경매와 일관성 유지
- 사용자 경험 향상
- 이미 경매 알림 시스템이 있어서 구현 쉬움

**구현 방법:**
- 경매 알림 시스템 재사용
- `useWishlistProjectMonitor` hook 생성
- 프로젝트 시작 알림 추가

**예상 시간:** 1-2시간

---

### 3순위: 통계/대시보드

**이유:**
- 프로젝트/경매 통계 표시
- 인기 프로젝트/경매 표시
- 사용자 참여도 향상

**구현 내용:**
- 성공률, 평균 후원금액
- 인기 프로젝트/경매
- 카테고리별 통계
- 차트 라이브러리 사용 (recharts 이미 설치됨)

**예상 시간:** 3-4시간

---

### 4순위: 소셜 기능 강화

- 공유하기 기능 실제 구현

**구현 내용:**
- 카카오톡 공유

- SNS 공유 메타 태그 (Open Graph, Twitter Card)

**예상 시간:** 2-3시간

### 5순위: 성능 최적화

**이유:**
- 사용자 경험 향상
